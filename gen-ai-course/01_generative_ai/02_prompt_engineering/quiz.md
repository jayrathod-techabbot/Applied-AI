# Quiz: Prompt Engineering

## Questions

### Question 1
What is "few-shot learning" in prompt engineering?

A) Training the model with few examples
B) Providing examples in the prompt
C) Using a small model
D) Learning in few seconds

### Question 2
Which technique encourages step-by-step reasoning?

A) Role prompting
B) Chain of thought
C) Zero-shot
D) Template prompting

### Question 3
What is the primary benefit of role prompting?

A) Makes the model faster
B) Sets context and expertise level
C) Reduces API costs
D) Increases creativity

### Question 4
What does "temperature" control?

A) Output length
B) Randomness/creativity
C) Processing speed
D) Token limit

### Question 5
When should you use structured output formats?

A) Always
B) When you need parseable results
C) Never
D) Only for code

### Question 6
What is a system prompt used for?

A) Setting up the API
B) Defining persistent model behavior
C) Error handling
D) Authentication

### Question 7
What happens with vague prompts?

A) Better outputs
B) Unpredictable outputs
C) Faster outputs
D) Shorter outputs

### Question 8
How should you iterate on prompts?

A) Write once and never change
B) Refine based on output quality
C) Always make them longer
D) Always use more examples

## Answers

1. B - Few-shot provides examples in the prompt
2. B - Chain of thought encourages reasoning
3. B - Role prompting sets context and expertise
4. B - Temperature controls randomness
5. B - Use structured output for parseable results
6. B - System prompt defines persistent behavior
7. B - Vague prompts lead to unpredictable outputs
8. B - Iterate and refine based on outputs
